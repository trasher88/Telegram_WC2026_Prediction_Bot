# WC2026 Prediction Bot — руководство по функциям

Telegram-бот для турнира прогнозов на Чемпионат мира 2026.

Бот умеет:

- показывать расписание матчей по стадиям;
- принимать прогнозы пользователей;
- блокировать прогнозы перед началом матча;
- автоматически синхронизировать матчи и результаты через football-data API;
- считать очки;
- показывать leaderboard;
- показывать форму игрока и персональную статистику;
- отправлять дневные уведомления;
- отправлять результаты прогнозов после завершения матчей;
- давать администраторам инструменты управления и диагностики.

## `/start` и `/rename`

При первом `/start` бот просит ввести имя игрока. Это имя сохраняется как `display_name` и используется в leaderboard, отчётах и рассылках.

`/rename` позволяет изменить турнирное имя в любой момент.

---

## `/matches`

Показывает расписание матчей ЧМ-2026.

Сначала бот показывает inline-кнопки стадий:

- групповой этап по турам;
- 1/16 финала;
- 1/8 финала;
- 1/4 финала;
- 1/2 финала;
- матч за 3 место;
- финал.

В расписании отображаются:

- время по Москве;
- флаги;
- русские названия команд;
- статус матча;
- итоговый счёт завершённых матчей.

Если пары плей-офф ещё не определены, матч может отображаться как `Неизвестная команда — Неизвестная команда`. Такие матчи видны в расписании, но недоступны для прогноза.

---

## `/predict`

Сценарий:

1. пользователь выбирает стадию;
2. выбирает матч;
3. выбирает счёт кнопкой или вводит вручную;
4. прогноз сохраняется в БД.

Особенности:

- прогноз можно изменить до закрытия приёма;
- повторный прогноз обновляет предыдущий;
- прогнозы закрываются за 5 минут до начала матча;
- матчи с неизвестными командами недоступны для прогноза;
- если scheduler не успел закрыть матч, проверка дедлайна сработает при сохранении.

Пример:

```text
✅ Прогноз сохранён
Бразилия 🇧🇷 - Марокко 🇲🇦 2:1
```

## `/my_predictions`

Показывает прогнозы пользователя.

Для каждого прогноза отображается:

- матч;
- прогноз пользователя;
- статус матча;
- итоговый счёт, если матч завершён;
- начисленные очки.

Статусы результата:

- `✅ Точный счёт (+2)`;
- `🟡 Исход (+1)`;
- `❌ Не сыграл`;
- `⏳ Матч ещё не завершён`.

---

## `/leaderboard`

Формат строки:

```text
🥇 Ванек - 7 очков (2) (прогнозов: 23)
```

Где:

- `7 очков` — общее количество очков;
- `(2)` — количество точных счетов;
- `(прогнозов: 23)` — общее количество прогнозов.

Сортировка:

1. больше очков;
2. при равных очках выше тот, у кого больше точных счетов;
3. при полном равенстве используется стабильная сортировка по ID.

---

## `/form`

Показывает форму игрока за последние 10 завершённых прогнозов.

Пример:

```text
📈 Форма игрока
Последние 10 завершённых прогнозов:

✅ 🇲🇽 Мексика — 🇿🇦 ЮАР 3:0
   Прогноз: 3:0 · +2 очка

🟡 🇧🇷 Бразилия — 🇲🇦 Марокко 2:1
   Прогноз: 1:0 · +1 очко

──────────────
🎯 Точные счета: 1
🏆 Очки за форму: 3 из 4
📊 Средний балл: 1.50
```

## `/my_stats`

Показывает персональную карточку игрока:

- очки;
- точные счета;
- угаданные исходы;
- промахи;
- всего прогнозов;
- завершённые прогнозы;
- ожидающие результата;
- очковая эффективность;
- прогнозная точность;
- точность счетов;
- средний балл;
- текущая серия.

Пояснения:

- очковая эффективность — набранные очки от максимально возможных;
- прогнозная точность — процент прогнозов, где игрок получил хотя бы 1 очко;
- точность счетов — процент точных счетов;
- средний балл — среднее количество очков за завершённый прогноз.

---

## Система очков

| Результат прогноза | Очки |
|---|---:|
| Точный счёт | 2 |
| Правильный исход | 1 |
| Неправильный прогноз | 0 |

Пример: матч закончился `2:1`.

| Прогноз | Результат |
|---|---|
| `2:1` | 2 очка |
| `1:0`, `3:2`, `2:0` | 1 очко |
| `1:1`, `0:1` | 0 очков |

---

## Плей-офф, дополнительное время и пенальти

Если матч дошёл до серии пенальти, победителю серии добавляется `+1` гол.

Примеры:

```text
После игры: 1:1
Первая команда выиграла по пенальти
Для прогнозов считается: 2:1
```

```text
После игры: 1:1
Вторая команда выиграла по пенальти
Для прогнозов считается: 1:2
```

То есть для расчёта прогнозов матч считается победой одной из команд, а не ничьей.

---

## Блокировка прогнозов

Прогнозы закрываются за 5 минут до начала матча.

Блокировка реализована в двух местах:

1. scheduler ставит `locked=1` за 5 минут до матча;
2. `services/predictions.py` дополнительно проверяет дедлайн при сохранении прогноза.

## Игровой день и уведомления

Игровой день:

```text
13:00 МСК текущего дня — 12:59 МСК следующего дня
```

Дневные уведомления:

| Время МСК | Кому | Что отправляется |
|---|---|---|
| 13:00 | всем игрокам | список матчей игрового дня |
| 18:00 | игрокам с пропущенными прогнозами | напоминание поставить прогнозы |
| 18:30 | администраторам | отчёт: кто поставил, кто не поставил, кому чего не хватает |

## Синхронизация матчей

Матчи синхронизируются с football-data API.

Scheduler:

- получает список матчей;
- обновляет команды, время, статус, стадию, тур и счёт;
- планирует lock jobs;
- обрабатывает завершённые матчи;
- отправляет результаты прогнозов.

Автоматическая синхронизация работает только если:

```env
ENABLE_API_SYNC=1
```

Для локальных тестов:

```env
ENABLE_API_SYNC=0
DB_PATH=test_wc2026.db
```
