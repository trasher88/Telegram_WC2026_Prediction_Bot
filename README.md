# Telegram WC2026 Prediction Bot

Telegram-бот для турнира прогнозов на Чемпионат мира 2026.

Проект рассчитан на private-friendly турнир между участниками: игроки ставят прогнозы на матчи, бот закрывает приём прогнозов перед стартом матчей, считает очки, отправляет результаты и показывает статистику.

## Основные возможности

- регистрация игрока с турнирным именем при первом `/start`;
- изменение имени через `/rename`;
- расписание матчей по стадиям и турам через `/matches`;
- прогнозы через `/predict` с быстрыми кнопками счёта и ручным вводом;
- запрет прогнозов на матчи с неизвестными командами;
- автоматическая блокировка прогнозов за 5 минут до начала матча;
- автоматический и ручной подсчёт очков;
- таблица лидеров с тай-брейком по точным счетам;
- персональная форма игрока за последние 10 завершённых прогнозов;
- персональная статистика игрока;
- дневные уведомления по игровому дню;
- ежедневные итоги прошедшего игрового дня;
- админ-команды для диагностики, ручного счёта, пересчёта очков и рассылок;
- опциональный Cloudflare Worker proxy для Telegram Bot API, если сервер не может ходить напрямую к `api.telegram.org`.

## Система очков

| Результат прогноза | Очки |
|---|---:|
| Точный счёт | 2 |
| Угадан исход матча | 1 |
| Прогноз не сыграл | 0 |

Пример: матч закончился `2:1`.

| Прогноз | Очки |
|---|---:|
| `2:1` | 2 |
| `1:0`, `3:2`, `2:0` | 1 |
| `1:1`, `0:1` | 0 |

Пример ничьей: матч закончился `2:2`.

| Прогноз | Очки |
|---|---:|
| `2:2` | 2 |
| `1:1`, `3:3`, `0:0` | 1 |
| `2:1`, `1:2` | 0 |

## Плей-офф и серия пенальти

Для матчей на вылет используется турнирное правило:

- если серия пенальти не проводилась, используется обычный итоговый счёт;
- если матч дошёл до серии пенальти, победителю серии добавляется `+1` гол;
- пример: после игры `1:1`, первая команда выиграла по пенальти — для прогнозов считается `2:1`;
- пример: после игры `1:1`, вторая команда выиграла по пенальти — для прогнозов считается `1:2`.

Эта логика применяется в `scheduler.py` функцией `extract_tournament_score()` до записи результата в таблицу `matches`.

## Игровой день

Игровой день считается по Москве:

```text
13:00 МСК текущего дня — 12:59 МСК следующего дня
```


## Уведомления

| Время МСК | Получатели | Содержание |
|---|---|---|
| 13:00 | Все игроки | Итоги прошедшего игрового дня. |
| 14:00 | Все игроки | Расписание текущего игрового дня. |
| 18:00 | Игроки с пропущенными прогнозами | Напоминание поставить прогнозы. |
| 18:30 | Администраторы | Отчёт: кто поставил, кто не поставил, кому чего не хватает. |

Дополнительно бот отправляет:

- сообщение о закрытии прогнозов на конкретный матч;
- результаты прогнозов после завершения конкретного матча.

## Итоги игрового дня

Ежедневная рассылка в 13:00 показывает:

- сыгранные матчи прошедшего игрового дня;
- очки игроков за этот игровой день;
- количество точных счетов;
- игрока дня;
- текущий топ общей таблицы.


Итоги отправляются только если все матчи прошедшего игрового дня завершены и имеют счёт.

## `/leaderboard`

Строка игрока выглядит так:

```text
🥇 Игрок - 7 очков (2) (прогнозов: 23)
```

Где:

- `7 очков` — общее количество очков;
- `(2)` — количество точных счетов;
- `(прогнозов: 23)` — количество прогнозов.

Сортировка:

1. больше очков;
2. при равных очках — больше точных счетов.

## `/form`

Показывает последние 10 завершённых прогнозов игрока:

- итог матча;
- прогноз игрока;
- начисленные очки;
- точные счета за форму;
- очки за форму;
- средний балл.

## `/my_stats`

Показывает персональную карточку игрока:

- очки;
- точные счета;
- угаданные исходы;
- промахи;
- прогнозы всего;
- завершённые прогнозы;
- прогнозы, ожидающие результата;
- очковая эффективность;
- прогнозная точность;
- точность счетов;
- средний балл;
- текущая серия.

Пояснения:

- очковая эффективность — сколько очков набрано от максимально возможного количества;
- прогнозная точность — процент завершённых прогнозов, где игрок получил хотя бы 1 очко;
- точность счетов — процент точных счетов от завершённых прогнозов;
- средний балл — среднее количество очков за один завершённый прогноз.

## Прогнозы на неизвестные команды

Матчи плей-офф могут заранее приходить из API без известных команд. Такие матчи:

- показываются в `/matches` и `/match_ids` как часть расписания;
- не появляются в `/predict`;
- дополнительно блокируются на уровне `services/predictions.py`, если пользователь нажмёт старую inline-кнопку.

## Cloudflare Worker для Telegram API

Если сервер не может напрямую подключиться к `api.telegram.org`, можно использовать Cloudflare Worker как proxy:

```text
Timeweb VPS → Cloudflare Worker → api.telegram.org
```

Код Worker:

```javascript
export default {
  async fetch(request, env, ctx) {
    const url = new URL(request.url);

    if (!url.pathname.startsWith("/bot")) {
      return new Response("Not found", { status: 404 });
    }

    const targetUrl = new URL(request.url);
    targetUrl.protocol = "https:";
    targetUrl.hostname = "api.telegram.org";
    targetUrl.port = "";

    const proxiedRequest = new Request(targetUrl.toString(), {
      method: request.method,
      headers: request.headers,
      body: request.body,
      redirect: "follow",
    });

    return fetch(proxiedRequest);
  },
};
```

Проверка:

```bash
curl -sS --connect-timeout 10 --max-time 30 "$TELEGRAM_API_BASE_URL/bot$BOT_TOKEN/getMe"
```

Ожидаемый ответ:

```json
{"ok":true,"result":{...}}
```

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

set -a
source .env
set +a

python -m compileall -q main.py config.py db.py scheduler.py api_client.py handlers repositories services states utils
python main.py
```

На Windows PowerShell переменные задаются через `$env:...`.

## Запуск через systemd

Пример unit-файла:

```ini
[Unit]
Description=Telegram WC2026 Prediction Bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/root/wc2026/test/Telegram_WC2026_Prediction_Bot
EnvironmentFile=/root/wc2026/test/Telegram_WC2026_Prediction_Bot/.env
ExecStart=/root/wc2026/test/Telegram_WC2026_Prediction_Bot/.venv/bin/python /root/wc2026/test/Telegram_WC2026_Prediction_Bot/main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Команды:

```bash
systemctl daemon-reload
systemctl enable wc2026-bot
systemctl start wc2026-bot
systemctl status wc2026-bot
journalctl -u wc2026-bot -f
```
